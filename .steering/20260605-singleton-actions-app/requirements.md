# 要求内容: シングルトンのアクションアプリを直接提供する

## 背景・目的

`actions-app` はシングルトン運用が前提である（`defaction` は常にグローバルな
アクションアプリ 1 個へ暗黙登録し、利用者が複数インスタンスを使い分ける
設計ではない）。しかし現状の API は、このシングルトン前提と噛み合っておらず、
二重構造になっている。

1. グローバル変数 `*app*` はライブラリのロード時に既に自動生成される
   （`src/main.lisp` の初期化ブロック）。つまり `defaction` は何もしなくても
   暗黙の `*app*` に登録できる。
2. にもかかわらず README / 機能設計は `(make-actions-app)` を利用者に呼ばせ、
   その戻り値を別変数（`*actions*`）に保持して **それを** mount させている。
3. `make-actions-app` は呼ぶたびに **新しいインスタンスを生成して `*app*` を
   上書き** する。ロード時に作られた初期インスタンスは捨てられる。

結果として利用者は「`*app*` に暗黙登録される世界」と「自分で生成・保持した
インスタンスを mount する世界」という、つながりの見えにくい二つを扱わされる。
`(make-actions-app)` を呼ぶ／呼ばないで挙動が変わる暗黙の依存も生じ、
シングルトンという単純な実態に対して API が過剰に複雑になっている。

本作業では、**ライブラリが用意したシングルトンを `*actions-app*` として直接
提供** し、利用者はインスタンス生成を意識せず、それをそのまま使う（`defaction`
で定義し、`*actions-app*` を mount する）形に整理する。

## ユーザーストーリー

- 開発者として、アクションアプリのインスタンスを自分で生成・保持したくない。
  ライブラリが用意したシングルトンをそのまま `defaction` の登録先・mount 対象
  として使いたい。
- 開発者として、`make-actions-app` を呼ぶ／呼ばないで挙動が変わる、という
  暗黙の前提に悩まされたくない。

## 受け入れ条件

- **AC1**: ライブラリは公開シンボル `*actions-app*` を提供する。これは
  ロード時に確定する単一の `actions-app` インスタンス（単一ルート登録済み）で
  ある。利用者は何も生成せずこれを使える。

  ```lisp
  (na:defaction like :post (params) ...)          ; *actions-app* に登録される
  (lack:builder (:mount "/actions" na:*actions-app*) *web*)
  ```

- **AC2**: `defaction` は `*actions-app*` に暗黙登録する。

- **AC3**: 旧変数名 `*app*` は廃止し、`*actions-app*` にリネームする
  （pre-1.0 のため破壊的変更を許容）。`*app*` は公開シンボルとして残さない。

- **AC4**: `make-actions-app` は **公開 API から外す（unexport）**。
  シングルトン生成のための内部コンストラクタとして残すが、利用者向け API
  ではなくなる。README / 公開 API 表からも除く。

- **AC5**: `make-actions-app`（内部）は **副作用を持たない純粋なコンストラクタ**
  にする。`setf *actions-app*` のようなグローバル書き換えは行わず、生成した
  インスタンスを返すだけにする。`*actions-app*` の確定はその戻り値を使った
  変数定義側で行う。

- **AC6**: テストは `*actions-app*` を `let` で再束縛し、内部の
  `make-actions-app`（`ningle-actions/app` から参照）で生成した隔離インスタンス
  に差し替えて従来どおり隔離できる（NFR3）。

- **AC7**: ディスパッチ・レジストリ・エンドポイント URL の挙動は不変。
  既存の機能テスト（404 / 405 / 正常・URL 安定性・クエリ付加）はすべて
  グリーンを維持する。

## 制約事項

- ライブラリの基本方針「薄さ最優先」を維持する。新たな依存は追加しない。
- 接頭辞 `/actions` 固定、mount は利用者責務、という方針は変更しない。
- 変更は「アクションアプリの提供方法・命名」に閉じ、アクションの登録・
  ディスパッチ・エンドポイント組み立てのロジックには手を入れない。
- 破壊的変更（`*app*` 廃止・`make-actions-app` 非公開化）を含むため、
  バージョンを上げる（0.2.0 → 0.3.0）。
